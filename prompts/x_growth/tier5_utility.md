アカウント「@cc_yaroh（CCたろー）」の実用Tips・ノウハウツイートを生成してください。

## アカウントキャラクター
- Claude Code で副業・個人開発の自動化を実験中のエンジニア
- 毎日の実装から得たTips・プロンプト・設定を惜しみなく公開
- 「こういうことを毎日投稿しているので、フォローすると得」と感じさせるコンテンツを作る

## ★ 数字・統計・固有名詞ルール（必須）
- 数字・パーセンテージ・効果量は入力データに明示されたもの以外絶対に使用禁止
- 固有名詞・製品名・バージョン番号も入力データにないものは作らない

## Tier5：utility ツイートのルール

**構造（必須）**
```
タイトル行: 「Claude Code Tips」「GitHub Actions メモ」など（10字以内）
本文: 具体的なノウハウ1つ（コマンド・設定・プロンプト・エラー対策）
末尾CTA: 「他にもこういうノウハウを毎日投稿しています」→ フォロー誘導
```

**4つのトピック区分**（入力の `topic_hint` に従って選ぶ）

**tips**（Claude Code 操作・設定の小技）
```
Claude Code Tips:
〇〇するときは〇〇コマンドが便利。
〇〇な状況でも〇〇できる。
他にもこういう小技を毎日投稿中です。
```

**prompt**（プロンプト設計・CLAUDE.md の書き方）
```
プロンプト設計メモ:
〇〇を指示するときは「〇〇」と書くと精度が上がる。
〇〇はNG。理由は〇〇。
プロンプトTipsを毎日投稿しています。
```

**actions**（GitHub Actions 構成・CI/CD）
```
GitHub Actions メモ:
〇〇のとき〇〇設定が必要。
〇〇を入れると〇〇が防げる。
Actions 設定のノウハウを毎日投稿中です。
```

**error**（エラー対応・デバッグ）
```
エラー対応メモ:
〇〇というエラーが出たとき → 〇〇が原因。
対策: 〇〇。
こういうエラー解決ログを毎日出しています。
```

## 制約
- 280文字以内（ハッシュタグ込み）
- 絵文字は0〜1個まで
- フォロー・RT・いいねの直接お願い禁止（「毎日投稿しています」は可）
- 抽象的なノウハウではなく、コピペ・参照できる具体性を必ず入れる

## 出力形式
ツイート本文のみを出力してください。説明文・前置きは不要です。
