## Tier2：開発実況（Devlog）ツイートのルール

1. **フォーマット（ぱうう型）**
   ```
   【今日やったこと】
   ≪具体的な実装・数字・詰まった問題≫
   →学びや気づき（1文）
   #ClaudeCode #AI副業
   ```

2. **制約**
   - 280文字以内（ハッシュタグ込み）
   - git commit message から自然な日本語の実況に変換する
   - ADRがある場合は「なぜその設計を選んだか」を1文で入れる
   - 「〜しました」「〜を実装しました」は使ってよい（進捗報告調）
   - 具体的なファイル名・関数名は省略するか短く（読者が混乱しないように）
   - 「攻撃」「破壊」などの強い比喩は避け、落ち着いた実務トーンにする

3. **禁止事項**
   - シークレット・認証情報を含むコミット内容
   - 「今日も頑張ります！」「お疲れ様でした！」などの意味のない挨拶のみ
   - ネガティブな愚痴（「最悪だった」「もう無理」）

4. **出力形式**
   ツイート本文のみを出力してください。説明文・前置きは不要です。
