# @cc_yaroh X 運用設計書

## このアカウントは何をしているか

**「Claude Code で副業を全自動化する過程を build in public する」** エンジニアとしてのキャラクターで、AI・開発ツール・副業収益化に関心を持つ層へのリーチを目指す。

投稿から KPI 記録まで、**人間が操作しなくても毎日自動で動く**仕組みを構築済み。

---

## 投稿コンテンツの設計

### 3つのピラー構成

1日10本の投稿を、テーマ別に3つのピラーで配分している。

| ピラー | テーマ | 1日の本数 | 狙いたい読者 |
|--------|--------|-----------|------------|
| **trend** | Claude / AI の最新ニュースを解説 | 4本 | AI・生成AIに関心がある層全般 |
| **devlog** | 実際の開発ログ・試行錯誤の記録 | 4本 | エンジニア・Claude Code ユーザー |
| **revenue** | 副業・フリーランス収益化の実況 | 2本 | 副業・AI活用で稼ぎたい層 |

### 投稿フォーマット (ぱうう型)

各ツイートは以下の4段構成で生成される。

```
① 肩書き宣言   — 「Claude Code で副業を全自動化中のエンジニアです」
② 数字 / 事実  — 具体的な情報・ファクト (ソースに存在するもののみ)
③ 解説         — 自分の視点・経験談を添える
④ CTA          — 質問・共感・フォロー誘導
```

数値の捏造は品質・信頼性を損なうため、AIへの指示で「入力ソースにない数字・効果量は絶対に使用禁止」と明示している。

## フォロワー獲得戦略

### 課題：コールドスタート問題

X のアルゴリズムは「自分のフォロワーにしかツイートを届けない」構造のため、0フォロワーの状態ではハッシュタグ経由の偶発的な発見しか期待できない。

### 打ち手：他アカウントへの返信でリーチを稼ぐ

毎朝、#ClaudeCode に関する人気ツイート (いいね2件以上) を自動収集し、返信文の下書きを生成。**ユーザーが内容を確認して気に入ったものを手動投稿**する運用 (1日15分の目安)。

他アカウントの投稿への返信は「そのアカウントのフォロワー全員に表示される可能性がある」ため、0フォロワーでも露出を稼ぐことができる唯一の近道。

返信はキャラクターを維持しながら、相手への共感・価値追加・経験談のいずれかで構成されるよう設計している。

### 週次スレッドで拡散力を高める

毎週日曜の夜、1週間の活動をまとめたスレッド (6〜7ツイート) を自動投稿する。スレッド形式は単発ツイートより RT・引用されやすく、まとめアカウントに取り上げられる可能性もある。

## 運用上の人間の作業

完全自動化を目指しているが、現時点で人の判断が入る箇所が1つある。

| 作業 | 頻度 | 所要時間 |
|------|------|----------|
| 返信下書きの確認・投稿 | 毎朝 | 約15分 |

それ以外 (投稿・KPI記録・スレッド生成) はすべて自動。
